### `User` объекты  
  
Пользовательские объекты являются ядром системы аутентификации. Обычно они представляют людей, взаимодействующих с вашим сайтом, и используются для включения таких функций, как ограничение доступа, регистрация профилей пользователей, связывание контента с создателями и т. д. В системе аутентификации Django существует только один класс пользователей, то есть «суперпользователи» или «администраторский персонал». пользователи — это просто пользовательские объекты со специальным набором атрибутов, а не разные классы пользовательских объектов.

# Authenticate
<h3>следует знать что:</h3>
**1. 
	Метод используется для проверки набора учетных данных.
	

**2. 
	Метод сравнивает входящие данные при регистрации с имеющимися данными в базе данных. 
	Если данные о пользователе сходят с входящими метод вернет объект [User]()

**3.
	 Метод принимает учетные данные в качестве аргументов: имя пользователя и пароль для случая по умолчанию.

**4. 
	проверяет их на соответствие каждому серверу аутентификации
