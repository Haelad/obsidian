# Logging
<h3>следует знать что:</h3>
**1. 
	Логирование - это фиксация состояния приложения в определенный момент времени. Оно позволяет понимать, что система работает как ожидается (и вообще работает), а также облегчает диагностику проблем

**2. 
	Нужно создать переменную, которая унаследует функции логирования.

**3.
	дополнительная настройка логирования позволяет увидеть время, модуль, уровень  важности логирования, и место куда эти 'логги' попадут (в файл, документ и т.д)

**4.
	Уровни важности событий:
	DEBUG - самая подробная информация, нужна только разработчику и только для отладки, например значения переменных, какие данные получены и т.д.
	INFO - информационные сообщения, как подтверждение работы, например запуск сервиса WARNING - еще не ошибка, но уже надо посмотреть - мало места на диске, мало памяти, много созданных объектов и т.д.
	ERROR - приложение еще работает и может работать, но что-то пошло не так
	CRITICAL - приложение не может работать дальше.

#### Синтаксис:
```python
from logging import getLogger, basicConfug, DEBUG, ERROR, FileHandler, StreamHadler

logger = getLogger(__name__)
FORMAT = '%(asctime)s : %(name)s : %(levelname)s : %(message)s'

file_handler = FileHandler("data.log")
file_handler.setLevel(DEBUG)

stream = StreamHadler()
stream.setLevel(ERROR)

basicConfig(level=DEBUG, format=FORMAT, hadlers=[file_handler, stream])
```
### Параметры:
##### Format:
- `FORMAT` - формат вывода информации для логгера
- `asctime` - фиксация времени "события" 
- `name` - фиксация имени логгера
- `levelname` - фиксация важности "события" 
- `message` - фиксация сообщения  
##### Handlers:
- `FileHandler` - обработчик событий, который будет записывать события в файл
- `StreamHadler` - обработчик событий, который будет записывать события в консоль
- `file_handler = FileHadler(" ")`- в скобках указывается названия файла 
- `file_handler.setLevel(DEBUG)` - в скобках  устанавливает уровень важности события, который он будет записывать в файл
##### Config:
- `basicConfig` - конфиг для логгера
- `level` - уровень важности события 
- `format` - формат которым логгер будет пользоваться
- `handlers` - обработчик событий, который будет записывать события 


> [!Примечание] Примечание
> `basicConfig` необязательно каждый раз создавать в ручную, его можно прочитать с помощью json для этого можно воспользоваться функцией logging.config

### пример с кодом:
```python
import logging.config

with open("logging.conf") as file:
	config = json.load(file)

logging.config.dictConfig(config)
logger = getLogger()
```
> здесь мы открыли наш конфиг с параметрами логирования и при помощи функции - `logging.config.dictConfig(config)`и прочли конфиг
