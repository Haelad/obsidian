# Что такое Celery?
Celery (Distributed Task Queue) - **это распределить очереди задач**, что значит что задачи не привязаны к определенному устройству и их можно перенести на другой компьютер.
# Какая основная задача Celery?
Celery нужен для того что бы разгрузить основной сервер от сложных и разных задач. 
С его помощью мы можем создать задачу, а затем развернуть ее на другом сервере.
# Как работает Celery?
С основного сервера мы отправляем сообщение, которое сообщает другому серверу начать выполнять эту задачу. Общение между серверами осуществляются при помощи формы очереди
## Что такое Очередь?
Очередь это место где хранятся сообщения, а приложение которое обеспечивает такую очередь, называется брокером.
## Что такое рабочий Celery?
Рабочий Celery - это отдельная единица экземпляра Celery, работающий на машине.
Основным процессом рабочего является - считывание данных из очереди брокера и распределения работы между дочерними процессами.

Дочерние процессы могут собираться в Pool, то есть в некий бассейн, которым можно будет управлять.


### Сторона отправляющие сообщения называют - Продюсер

### Сторона получающие сообщения и выполняющие задачи называют -  Потребитель






# Celery Beat
Celery может выполнять задачи в определенное время.

Для этого нужно запустить экземпляр celery Beat
```python
'collect_statisstics': {
	'task': 'path.to.your.task'
	'schedule': crontab(hour=12, minute=0),
},
```




# Для чего используется Celery?
Celery обычно используется для:
- отправки электронных писем;
- отправки уведомлений;
- создания отчетов;
- Что то, что заставляет клиента ждать;
